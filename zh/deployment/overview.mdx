一般来说，您可以将应用程序部署到任何支持 Node.js 的托管服务提供商。在本文档中，我们涵盖了从 Vercel 等无服务器平台到使用 Docker 自托管解决方案的多种部署目标。

由于 Next.js 是由 Vercel 构建的框架，当您部署到 Vercel 时，将获得最佳的开发体验。

## 项目设置

要在任何托管服务提供商上部署 SuperStarter，您需要为 `app`、`api` 和 `web` 应用创建 3 个新项目。选择您的仓库后，将 Root Directory 选项更改为所选应用，例如 `apps/app`。这应该会自动检测 Next.js 设置，从而确定构建命令和输出目录。

然后，将所有环境变量添加到项目中。

最后，只需点击"Deploy"，Vercel 将处理其余工作！

<Note>
这里我们假设您已经了解这些应用程序之间的区别以及您将要部署哪些应用程序
</Note>

## 环境变量

我们建议在部署前利用托管服务提供商的团队环境变量功能。库使用的变量需要存在于所有包中，重复它们可能会很麻烦。了解更多关于 SuperStarter 中[环境变量](/zh/env)的工作原理。

<Note>
首次部署可能会失败

如果环境变量设置不正确，第一次部署可能会失败。这很正常。让第一次部署失败，然后修正环境变量，重新部署。
</Note>
